# Динамическая генерация DAG-ов xStream Export

Данная директория содержит инструменты для миграции и работы с динамическими DAG-ами выгрузки данных через xStream (Airflow).

## Структура файлов

- **`xs_export.py`** — Основная фабрика DAG-ов. Содержит логику генерации Airflow DAG на основе JSON-метаданных и встроенные утилиты для сборки SQL.
- **`xs_common.py`** — Общая библиотека с операторами и стандартными функциями (включая `make_xs_export_task_group`).
- **`export_xs_optimized.json`** — Оптимизированные метаданные для 191 выгрузки. Содержит структуру таблиц, расписание и специфичные части SQL-запросов.
- **`parse_meta.py` / `optimize_meta.py`** — (Встроены в `xs_export.py`) Логика для извлечения и оптимизации метаданных из статических DAG-файлов.

## Как это работает

1. Airflow сканирует `xs_export.py`.
2. Скрипт читает `export_xs_optimized.json`.
3. Для каждой записи в JSON создается объект `DAG`.
4. SQL-запросы (`update_exp`, `load_delta`, `export_delta`) динамически собираются из общих колонок таблицы и специфичных условий (`JOIN`, `WHERE`, `FINAL`).
5. Задачи группируются через `make_xs_export_task_group`.

## Использование

### В Airflow
Для активации динамических DAG-ов убедитесь, что:
1. Эта директория доступна Airflow (находится в `dags_folder` или добавлена в `PYTHONPATH`).
2. В Airflow установлена переменная окружения или настроен импорт так, чтобы `xs_export.py` мог найти `xs_common.py`.

### Обновление метаданных
Если структура исходных таблиц или SQL-запросы изменились, необходимо обновить `export_xs_optimized.json`. Внутри `xs_export.py` сохранены функции `parse_sql` и `optimize_table`, которые позволяют перегенерировать метаданные программно.
